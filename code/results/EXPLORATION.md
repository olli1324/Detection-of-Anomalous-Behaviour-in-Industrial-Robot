## 1. Dataset shape

| split        | rows     | cols (with bookkeeping) | sensor features |
|--------------|----------|-------------------------|-----------------|
| KukaNormal   |   233792 | 87 (after aligning)     | 85 |
| KukaSlow     |    41538 | 87                      | 85 |

- The provided `KukaNormal.npy` is shipped with **86** columns (no
  trailing `anomaly`). The loader appends an all-zero `anomaly`
  column to align it with `KukaSlow`.
- Bookkeeping columns: `action` (integer action id) and `anomaly`
  (0/1 label). They are excluded from the feature set.

## 3. Degenerate channels

4 channels are **constant in KukaNormal**
(per-channel std ≤ 1e-9):

```
        - sensor_id2_temp
- sensor_id5_temp
- sensor_id6_temp
- sensor_id7_temp
```

These are all temperature readings on sensors 2, 5, 6, 7. **A
direct check showed each of these channels has exactly one unique
value in Slow as well**:

- sensor_id2_temp = sensor_id5_temp = 144.12 in both splits
- sensor_id6_temp = sensor_id7_temp = 180.24 in both splits

So these are dead/disabled sensors that emit a fixed value
regardless of operating regime. They carry **no information** for
the anomaly task. (The earlier impression that "z_shift was
infinite" came from fp32 rounding in the mean over ~234k identical
values producing a tiny non-zero numerator over a near-zero std,
which is just numerical noise.)

Decision: **drop them from the feature set** entirely. The AE will
train on 81 sensor channels.

## 4. Action distribution

- Both splits use a discrete `action ∈ {0, …, 31}` label.
- **32 distinct actions in Normal**, **31 in Slow**: action 9 is
  missing entirely from `KukaSlow`. Any per-action evaluation must
  handle that.
- Slow has fewer examples per action, but **action segments
  (consecutive timesteps with the same action) are 1.2–1.65×
  longer** in Slow than in Normal. This directly confirms the
  dataset description: the robot is executing the same trajectories
  but more slowly. See `results/figures/02_segment_lengths.png`.

## 5. Channel-level shift

See `results/figures/03_top_zshift.png` for the top-25 channels by
absolute mean shift between Slow and Normal (in units of Normal's
std). The dominant signals are:

- Some quaternion channels with large mean shifts (orientation
  coverage differs in Slow).
- The four constant-in-Normal temperature channels (z-shift is
  mathematically infinite, handled separately; see §3).
- Several electrical features (apparent_power, power_factor) show
  modest but consistent shifts.

## 6. Separability sanity check

Trained a logistic regression on a balanced 5 000 + 5 000 sample of
standardised feature vectors (degenerate channels excluded, no
windowing) with a 70/30 split:

- **Test accuracy: 0.981**
- **Test ROC-AUC: 0.996**

See `04_pca_scatter.png` for the PCA(2) projection of the test
sample.

Interpretation: the two distributions are highly separable already
in the raw feature space, before any temporal modelling. This sets
an *upper bound* on what an unsupervised AE could realistically
achieve, and tells us:

- The headroom for AE / AAE detection is large; both should clear
  this benchmark comfortably if our pipeline is correct.
- The interesting question for the AE-vs-AAE comparison is **not
  "can we beat chance?"** but **"does the adversarial regulariser
  give measurable improvements in calibration, ranking quality,
  and stability"**, and at what extra training cost.
- We should report both *threshold-free* metrics (ROC-AUC,
  PR-AUC) and *threshold-dependent* ones, and we should **not** use
  the logistic regression to choose AE thresholds (it's a "cheat":
  it sees Slow during training).

## 7. Decisions for the AE pipeline

Based on this exploration:

1. **Feature set**: drop the 4 dead temperature channels →
   **81 sensor channels** in the AE input.
2. **Standardisation**: per-channel `StandardScaler` fit *only on
   the Normal training split*. Stats are computed in fp64 to
   avoid the fp32 rounding pitfall observed during exploration.
3. **Windowing**: build sliding windows that respect action
   boundaries (windows that cross actions risk leaking the "slow"
   signal into structure rather than dynamics). Initial window
   length: 64 timesteps (≈ 70% of a typical Normal segment),
   stride 16.
4. **Splits**: segment-level, stratified by action. 70 % train,
   10 % validation (Normal only) for threshold calibration,
   20 % test. The entire Slow set is reserved for the final
   anomaly evaluation.
5. **Evaluation**: ROC-AUC, PR-AUC, F1 at the threshold that
   maximises validation F1 (or fixes a target FPR), plus
   per-action breakdown, and explicitly avoid the point-adjust
   trick called out in Kim et al. (2022).
